PRODUCT / 01
No-code · Hosted page

HyperCheckout

A fully pre-built, hosted payment page. Your backend creates a session, your app redirects the user — Juspay handles every pixel of the checkout UI and every step of the payment flow from there. Nothing for your team to build or maintain.

Zero UI development Fastest to market Limited UI control

Like renting a fully furnished checkout counter. You escort the customer to the door — Juspay handles everything inside. Payment form, OTPs, bank redirects, and the success screen are all theirs.

How payment flows
U
Customer clicks "Pay"On your website or app. No Juspay involvement yet.
M
Your backend creates a sessionServer-side POST /session call with order details. Juspay responds with a hosted payment URL.
J
User lands on Juspay's pageBrowser redirects to Juspay's hosted UI. Your code does nothing here — Juspay renders the full checkout.

Integration steps
1
Create a session (backend)

Call POST /session with order_id, amount, customer_id, return_url. Response contains a hosted payment URL.

2
Redirect the user

Point the browser to the URL from step 1. On mobile apps, open it inside an in-app WebView.

3
User pays on Juspay's page

Nothing to build. Juspay handles the full checkout UI, OTPs, and all bank redirect flows.

4
Verify on return

User lands on your return_url. Call GET /order/{id} server-side — never trust URL params alone.

5
Configure webhooks

Register your endpoint in the dashboard. Juspay sends HMAC-signed POST callbacks for every payment status change.

Express Checkout SDK

You build and own the entire payment UI. Juspay's SDK runs underneath — handling payment processing, UPI deep links, 3DS flows, and smart routing invisibly. Your design, your UX, Juspay's engine.

Full UI control No payment logic to build Requires UI development Mobile-first only

How payment flows
U
Customer taps "Pay" in your appOn your custom-built screen. Your design, your components.
M
Your UI calls an SDK methodYou call HyperServices.process(payload) with the selected payment method details.
J
SDK handles all payment logicJuspay's SDK communicates internally — UPI intent, collect flows, 3DS, bank OTPs — all invisible to your team.
B
Bank processes the paymentSDK manages the deep link or redirect. User sees the bank's native OTP screen if required.
M
SDK fires callback to your UIYour onEvent listener receives a payment_result. Your screen shows success or failure.
Integration steps
1
Add SDK to your project

Android: build.gradle. iOS: CocoaPods / SPM. React Native / Flutter: install the plugin package.

2
Initialise SDK on app start

Call HyperServices.initiate() with your merchant ID. Prefetches data to reduce checkout latency.

3
Create a session (backend)

Your server calls Juspay. Pass the client_auth_token from the response down to the SDK.

4
Build your payment screen

Design and code your UI — UPI input, saved cards, wallet options. 100% your team's work.

5
Call process() on submit

When user confirms, call HyperServices.process(payload) with the chosen method as JSON.

6
Handle event callbacks

Listen on onEvent for show_loader, hide_loader, payment_result. Update your UI accordingly.

7
Verify on your backend

Always confirm status via GET /order/{id} server-side before fulfilling the order.

Express Checkout API

Pure REST APIs. No SDK, no Juspay UI. You own the entire stack — screens, payment state machine, retry logic, error handling. Juspay is purely a backend payment processor you call over HTTPS.

Maximum control Platform agnostic High engineering effort Handle all edge cases

How payment flows
U
Customer fills your payment formEntirely your-built UI — every input, label, and button is yours.
M
Your backend calls Juspay REST APIServer-to-server POST /txns with payment method details. Juspay never touches your frontend.
J
Juspay routes to the best acquirerIf 3DS is needed, the response contains an authentication.url for your frontend to redirect to.
B
Bank processes · posts back to JuspayAfter auth, bank posts result to Juspay. Juspay fires a signed webhook to your endpoint.
M
Your backend receives webhook · fulfilsVerify HMAC signature. Call GET /orders/{id} to confirm. Then update order state.
Integration steps
1
Set up authentication

Get merchant_id + api_key from the dashboard. All calls use HTTP Basic Auth.

2
Create an order

Call POST /orders to register the order. This ties all payment attempts to a single trackable object.

3
Initiate a transaction

Call POST /txns. Payload varies: cards need number + CVV (or token), UPI collect needs VPA, netbanking needs bank code.

4
Handle 3DS / bank redirects

If response has authentication.url, redirect the user there. Juspay posts a webhook after the bank responds.

5
Configure webhook endpoint

Register your URL in the dashboard. Verify HMAC signature on every event before updating order state.

6
Confirm status & fulfil

Call GET /orders/{id} server-side — source of truth. Webhooks can arrive out of order.

7
Implement refunds

Call POST /orders/{id}/refunds. Partial refunds supported — pass any amount ≤ original charge.
