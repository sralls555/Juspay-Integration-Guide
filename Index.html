PRODUCT / 01 No-code · Hosted page

HyperCheckout

A fully pre-built, hosted payment page. Your backend creates a session, your app redirects the user — Juspay owns every pixel of the checkout UI and every step of the payment flow from there.

🏪

Think of it as: renting a fully furnished checkout counter. You escort the customer to the door. Juspay handles everything inside — payment form, OTPs, bank redirects, success screen.

Integration steps
1
Create a session (backend)

Call POST /session with order_id, amount, customer_id, return_url. Get back a hosted payment URL.

2
Redirect the user

Point the browser to the URL from step 1. On mobile apps, open inside an in-app WebView.

3
User pays on Juspay's page

Nothing for your team to build. Juspay renders checkout, handles OTPs, and manages all bank redirects.

4
Verify on return

User lands on your return_url. Always call GET /order/{id} server-side before fulfilling — never trust URL params alone.

5
Configure webhooks

Register your endpoint in the dashboard. Juspay sends HMAC-signed POST events for every status change.

PRODUCT / 02 SDK-based · Custom UI

Express Checkout SDK

You build and own the entire payment UI. Juspay's SDK lives underneath — handling payment processing, UPI deep links, 3DS flows, and smart routing invisibly behind your branded screens.

Think of it as: Juspay gives you the engine and gearbox. You design the entire car body around it. The engine (payment logic) is theirs — every screen the user sees is yours.

Integration steps
1
Add SDK to your project

Android: build.gradle. iOS: CocoaPods / SPM. React Native / Flutter: install the respective plugin package.

2
Initialise SDK on app start

Call HyperServices.initiate() with your merchant ID. Prefetches data to reduce checkout latency.

3
Create a session (backend)

Server calls Juspay. Pass the client_auth_token from the response down to your app's SDK layer.

4
Build your payment screen

Design and code your UI — UPI input, saved cards, wallet options. 100% your team's work. Juspay has no say here.

5
Call process() on submit

On user confirm, call HyperServices.process(payload) with the chosen method and details as JSON.

6
Handle SDK event callbacks

Listen on onEvent for show_loader, hide_loader, payment_result. Update your UI accordingly.

7
Verify on your backend

Always confirm status server-side via GET /order/{id} before fulfilling. Never trust frontend callbacks alone.

PRODUCT / 03 API-first · Full ownership

Express Checkout API

Pure REST APIs. No SDK, no Juspay UI. You own everything — screens, payment state machine, error handling, retries, UX. Juspay is purely a backend processor you call over HTTPS.

Integration steps
1
Set up authentication

Get merchant_id + api_key from dashboard. All calls use HTTP Basic Auth.

2
Create an order

Call POST /orders to register the order. Ties all payment attempts to a single trackable object.

3
Initiate a transaction

Call POST /txns. Payload varies: cards need number + CVV (or token), UPI collect needs a VPA, netbanking needs a bank code.

4
Handle 3DS / bank redirects

If response contains authentication.url, redirect your user there. Juspay receives the bank response and posts a webhook.

5
Configure webhook endpoint

Register your URL in the dashboard. Verify the HMAC signature on every incoming event before updating order state.

6
Confirm status, fulfil order

Call GET /orders/{id} server-side. This is the source of truth — webhooks can arrive out of order or be delayed.

7
Implement refunds

Call POST /orders/{id}/refunds. Partial refunds are supported — pass any amount ≤ original charge.
